一、项目定位与核心价值
念念·GBrain 是一个个人 AI 双循环知识生命体，核心理念是：

通过客观行为痕迹构建动态用户画像，并用结构化知识库持久化"编译真相"

它融合了三个关键思想：

念念的实时情境感知
GBrain的 Compiled Truth + Timeline 知识范式
LLM Wiki的异步编译思想

3.2 Compiled Truth + Timeline 模式
这是知识库最核心的设计创新：

维度	Compiled Truth	Timeline
性质	当前最佳理解	客观证据链
操作	可重写	只追加
作用	快速获取认知	追溯演变过程
类比	Wikipedia 词条	Git 提交历史

四、混合搜索实现（技术亮点）
4.1 双通道检索 + RRF 融合

Python

async def hybrid_search(query: str, limit: int = 20) -> list[dict]:
    # 并行执行两种搜索
    fts_results, vector_results = await asyncio.gather(
        search_fts(db, query, limit=limit * 2),      # FTS5 全文搜索
        _vector_search(db, query, limit=limit * 2),   # 向量相似度搜索
    )
    
    # RRF (Reciprocal Rank Fusion) 融合
    fused = _rrf_fusion(fts_results, vector_results, k=60)
    
    # 4层去重策略
    return _deduplicate(fused)[:limit]
4.2 四层去重策略
Slug 去重：同一页面只保留最佳 chunk
Jaccard 相似度去重：相似度 > 0.85 的文本块去重
类型比例控制：单一类型不超过总数的 60%
Slug 频次控制：每个 slug 最多出现 2 次
为什么用 RRF？

FTS 擅长精确匹配关键词
向量搜索擅长语义相似
RRF 公式 score = 1/(k+rank) 无需调参，对两种检索方式公平

5.1 信号提取（双通道）

async def tool_extract_signals(text: str) -> list[dict]:
    # 通道1：正则快速提取（消费、决策、情绪、冲突）
    regex_signals = extract_signals_regex(text)
    
    # 通道2：LLM 深度提取（决策犹豫、价值观、人际关系等）
    llm_signals = await extract_signals_llm(text)
    
    # 去重合并
    return deduplicate_signals(regex_signals + llm_signals)
信号类型设计：

expense - 消费行为

emotion - 情绪（positive/negative）
relationship - 人际关系变化
goal - 目标设定
reflection - 自我反思
